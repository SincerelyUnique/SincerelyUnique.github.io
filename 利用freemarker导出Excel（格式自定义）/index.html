遇到的问题：

  1. 使用 xml 作为 freemarker 模板时，一些含有特殊字符的变量会被 xml 转译，导致导出结果显示错误，此时可以使用 apache common lang3 包下的工具处理：

  1. 使用 freemarker 在模板中做日期格式处理时遇到问题，处理年月日时显示正常，但如果传值为 null 时，显示”1900/1/0“，改成了从后台使用 SimpleDateFormat 处理。

  2. 日期处理中时分秒的单独显示未成功，参考使用了以下 3 中解析方式，都没有解析成功，可能是其他地方错了，改成了从后台使用 SimpleDateFormat 处理。

${reviews.date?string('hh:mm:ss')}
${reviews.date?string('HH:mm:ss')}
${reviews.date?time('hh:mm:ss')}
  1. 给特定格式的日期做判空，为空时不显示，改成如下
${(reviews.createTime?string('yyyy-MM-dd'))!''}
